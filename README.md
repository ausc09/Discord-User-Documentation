# Discord User Documentation

This repository contains user instructions for setting up and using Discord for CST team collaboration (teams of 3–10 members).  
The documentation is written for beginner users and explains how to complete key collaboration tasks clearly and efficiently.

The instruction set covers:
1. Account setup, adding friends, and starting chat  
2. Server creation and configuration  
3. Channel organization and permission management  
4. Hosting team meetings on Discord  

---

## Methods and Process Used

The final documentation was developed using a structured technical writing process:

1. Defined audience and scope  
   - Target users: first-term CST students with little or no Discord experience  
   - Scope focused on practical academic collaboration tasks only  

2. Researched platform behavior and workflow expectations  
   - Reviewed Discord interface patterns and relevant support content  
   - Identified common user errors and friction points to address in instructions  

3. Planned task-based information architecture  
   - Organized content into separate procedural modules by user goal  
   - Sequenced tasks in onboarding order to reduce confusion  

4. Applied instructional writing best practices  
   - Used numbered, action-oriented steps  
   - Included rationale where decisions affect team workflow  
   - Added screenshots to support recognition and task completion  
   - Added troubleshooting and glossary support pages  

5. Applied web and readability best practices  
   - Used clear headings, chunked sections, and consistent terminology  
   - Maintained scannable structure for quick navigation  
   - Ensured internal linking through MkDocs navigation  

6. Built and validated with documentation tooling  
   - Authored in Markdown  
   - Built and previewed with MkDocs  
   - Resolved formatting, navigation, and file-reference issues  
   - Deployed as a static documentation site via GitHub Pages
