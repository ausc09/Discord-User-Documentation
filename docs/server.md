# Task 2: Create and Configure a Discord Server

## Brief Overview

In this task, you will create a new Discord server and configure the core settings needed for a safe, organized, and professional collaboration space.  
By the end, your server will have a proper identity, moderation baseline, onboarding-ready structure, and launch-ready settings.

!!! note "Audience"
    This guide is written for students or team members setting up a new server for group collaboration.

---

## Prerequisites

Before starting, confirm:
- You already have a Discord account and are logged in.
- You are using Discord desktop app or web version.
- You have a clear server purpose (class, team, project, community).

!!! tip "Planning first"
    Decide server name, icon, and purpose before setup to avoid rework later.

---

## Step 1: Open Discord and Start a New Server

1. Open Discord and sign in.
2. In the left sidebar, click the **+ (Add a Server)** button.
3. Select **Create My Own**.
4. Choose the server type that best matches your use case.

  <img src="images/task2-step1-create-start.png"
       alt="Discord add server dialog showing create server options"
       title="Start creating a new Discord server">

</figure>

---

## Step 2: Enter Server Name and Create

1. Type a clear server name.
2. Optionally upload a temporary icon.
3. Click **Create** to generate the server.

!!! note "Naming"
    Use a short and recognizable name so members can find it quickly.

---

## Step 3: Open Server Settings

1. Click the server name at the top-left.
2. Select **Server Settings** from the dropdown.
3. Confirm you are in the settings panel before continuing.

<figure>
  <img src="images/task2-step3-server-settings.png"
       alt="Discord server dropdown with server settings option"
       title="Open server settings for configuration">
  <figcaption>Figure 2. Server Settings is the control center for this task.</figcaption>
</figure>

---

## Step 4: Configure Server Identity in Overview

1. In **Overview**, confirm/update the server name.
2. Upload the final server icon.
3. Add a description if your Discord interface provides that option.
4. Click **Save Changes**.

<figure>
  <img src="images/task2-step4-overview.png"
       alt="Discord overview settings page with name and icon controls"
       title="Set server identity in Overview">
  <figcaption>Figure 3. Configure name and icon for a consistent server identity.</figcaption>
</figure>

---

## Step 5: Enable Community Mode (Recommended)

1. In **Server Settings**, click **Enable Community**.
2. Select **Get Started**.
3. Complete required setup checks.
4. Finish community setup and save.

!!! tip "Why this helps"
    Community mode unlocks additional server-level management and safety controls.

---

## Step 6: Configure Verification Level

1. Go to **Safety Setup** or **Moderation**.
2. Locate **Verification Level**.
3. Set an appropriate level (commonly **Medium** for team use).
4. Save changes.

!!! caution "Security baseline"
    Verification helps reduce spam and unauthorized activity.

<figure>
  <img src="images/task2-step6-verification.png"
       alt="Discord moderation settings with verification level options"
       title="Set verification level for server safety">
  <figcaption>Figure 4. Verification settings establish a basic trust gate for new members.</figcaption>
</figure>

---

## Step 7: Enable Content and Safety Filters

1. In moderation/safety settings, enable available media/content filtering.
2. Review anti-spam related options and enable appropriate defaults.
3. Save changes.

!!! note "Balanced setup"
    Start with moderate protections and adjust later if needed.

---

## Step 8: Set Default Notification Preference

1. Open server notification settings.
2. Set default notifications (commonly **Only @mentions**).
3. Save changes.

!!! tip "Member experience"
    Thoughtful defaults reduce notification fatigue for new members.

---

## Step 9: Configure Invite Link Policy

1. Click server name → **Invite People**.
2. Select **Edit Invite Link**.
3. Set:
   - Link expiration
   - Maximum number of uses
4. Generate/copy the invite link.

!!! warning "Access control"
    Avoid permanent unlimited links unless open access is intentional.

<figure>
  <img src="images/task2-step9-invite-link.png"
       alt="Discord invite link dialog showing expiration and max uses"
       title="Configure invite policy before sharing access">
  <figcaption>Figure 5. Invite link rules control how users can join.</figcaption>
</figure>

---

## Step 10: Final Server Configuration Review

1. Re-open **Server Settings** and verify each saved setting.
2. Confirm:
   - Server identity is correct
   - Community mode is enabled (if used)
   - Verification and safety settings are active
   - Notification default is set
   - Invite policy is configured
3. Record a screenshot of final settings for documentation.

!!! note "Completion check"
    At this point, server creation and core configuration are complete.

---

## Conclusion

You have successfully created and configured a Discord server with foundational server-level settings, including identity, safety, notifications, and invite controls.  
The server is now properly prepared for the next workflow stage.
